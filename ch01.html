奠定基础

// Laying the Groundwork — 所有位运算技巧的地基

在钻进那些令人叫绝的位操作技巧之前，Warren 先把舞台搭好： 计算机代数记号、RISC 指令集约定、二进制/十六进制、补码系统、 C 语言运算符优先级——这些看似无聊的准备工作， 却是整本书所有魔法得以成立的前提条件。

computer algebra RISC ISA two's complement operator precedence branch-free code notation
§1-1 记号系统 — Notation
// CORE CONCEPT 本书区分两套数学体系：普通算术（数学上的加减乘除）和计算机算术（比特串的有限域运算）。 计算机算术的结果总是被 mod 2ⁿ 截断，n 为机器字长。默认 32 位，有符号整数用二进制补码表示。

📐 粗体变量 = 寄存器内容

书中用粗体 x 表示寄存器中的比特串（无类型解释）， 用细体 x 表示其在某种解释（有符号/无符号）下的算术值。 这个区分极其重要——同一个 32 位模式 0xFFFFFFFF， 有符号解释是 −1，无符号解释是 4,294,967,295。

// 同一比特串，两种人格 unsigned u = 0xFFFFFFFF; // u = 4294967295 int s = 0xFFFFFFFF; // s = -1 // 两者底层 bits 完全一样！

⚡ 无分支优先原则

  • 阻碍 CPU 乱序执行

溢出陷阱

本书假设整数溢出中断已屏蔽（C/Java 正常行为）， Pascal/Ada 用户需要特别小心。溢出时结果按 mod 2³² 截断， 这正是位操作技巧能奏效的物理基础。

// 危险! 这在 C 中是 UB (未定义行为) int x = INT_MAX + 1; // ← UB in C // 安全: 用无符号运算 unsigned u = UINT_MAX + 1; // u = 0 ✓

§HIST 历史演变 — 位运算技术的前世今生
1940s
冯·诺依曼时代
EDVAC 首份草案（1945）确立了二进制、固定字长整数的基础。 Bell Labs 的 Stibitz 已在使用复数计算机，位操作从硬件层面诞生。 有符号数表示在原码、反码、补码之间争论不休。
1950s–60s
IBM 360 统一补码
IBM System/360 用补码作为唯一有符号整数表示， 终结了三种表示并存的混乱局面。Christopher Strachey 1961 年在 CACM 发表 "Bitwise Operations"，是位运算理论的早期奠基论文。
1972
HAKMEM — 位操作圣经诞生
MIT AI 实验室 Gosper、Beeler、Schroeppel 在 PDP-10 上写出 HAKMEM（AIM-239）， 那是一本奇异的"位操作技巧大全"。Guy Steele Jr. 读到它如获至宝， 后来他推动 Warren 把这类技巧系统整理成书。
1980s–90s
RISC 革命与编译器时代
MIPS、SPARC、PowerPC 等 RISC 架构普及， "每条指令都计数"的设计哲学催生了大量位操作优化。 GCC 编译器开始自动生成魔法数除法（magic number division）。 Martin Hopkins（"Mr. Compiler"）在 IBM 将每一个时钟周期都变成竞赛。
2002 / 2013
Hacker's Delight 第一/二版
Warren 在 IBM 工作数十年积累的技巧终于成书。 Guy Steele Jr. 撰写了 50 页书评并贡献了大量算法。 第二版（2013）新增了比特洗牌、SAG 操作、浮点技巧等内容。
2010s+
硬件指令化
x86 POPCNT、LZCNT、TZCNT、BMI1/BMI2 指令集把许多软件技巧直接烧进硬件。 ARM 的 CLZ、RBIT 同样如此。书中的"软件技巧"很多已经有了硬件快捷路径， 但算法思想永不过时。

§CRITIQUE 深度点评 — 技术哲学与批判性思考

补码：一个"偶然的完美"设计

为什么选补码，而非原码/反码？

1960 年代 IBM 360 选择补码并非偶然。补码让加法器不需要区分符号—— 同一套电路可以处理有符号和无符号整数，硬件成本直接砍半。 原码需要符号位单独处理，有两个零（+0 和 -0），反码同样有两个零且进位规则复杂。 补码的"意外收获"是让位操作技巧如此优雅：-x = ~x + 1 这个公式简单到像个数学笑话，但它是整本书所有魔法的物理基础。

有没有其他实现方式？

现代量子计算探索了超越二进制的三值逻辑（ternary）； Google 的 Brain Float（bfloat16）把精度和范围重新权衡。 但对于通用整数运算，补码在硬件简单性 × 数学完备性 × 工程实用性 三维度上的综合得分无出其右。唯一的"副作用"是 INT_MIN 的绝对值等于自身—— 这个边界条件在本书中反复出现，是很多技巧的细节注意点。

RISC vs CISC：这套书对 CISC 程序员还有用吗？

有，而且非常有用。即使在 x86 上，CPU 内部执行的也是微操作（µops）， 本质上是 RISC。更重要的是，算法思想不依赖具体ISA—— 理解"清除最低位1比特"的逻辑，无论你用 x86 的 BLSR 还是 RISC 的 and r, x, x-1，都是同一个道理。 CISC 指令集只是把常见模式封装成单条指令，不妨碍从算法层面理解问题。

可推广到什么地方？

模块化思维：Warren 把整个计算机世界抽象成"字长为 n 的有限域上的位操作"， 这套思维可以直接迁移到：密码学（有限域 GF(2ⁿ)）、 信号处理（定点数运算）、AI 量化（低比特推理）、硬件描述语言（VHDL/Verilog）。 凡是"有限精度整数"的地方，这本书就有用。 凡是"想要快速但不知道为什么"的地方，先把这章读透再说。

⚠️ 注意事项与局限性

这本书的前提假设是补码、32位字长、整数溢出不中断。 在 Rust 中 debug 模式整数溢出会 panic；在某些嵌入式平台字长不是32位； 在 C23 标准中有符号整数溢出的 UB 语义正在规范化。 此外，随着 SIMD/GPU 计算普及，"逐字操作"的假设正在被"向量化"取代—— 很多技巧需要适配 AVX-512 或 CUDA kernel 才能发挥最大价值。 读这本书时，脑子里要始终有一个问号：在我的目标平台上，这个假设成立吗？
