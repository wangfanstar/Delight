1972 · MIT AI Lab

Bill Gosper 在 HAKMEM（Memo 239）中提出此算法。那个年代的黑客们痴迷于用最少的 PDP-10 指令完成最复杂的事。这不是"优化"，这是一种信仰 —— 在硬件资源极度匮乏时，每一条指令都是战场上的子弹。

本质

补码的数学结构被 Gosper 当作一台精密的齿轮组来使用：x & (-x) 提取最低位的 1 —— 这不是技巧，而是补码系统"进位链溢出"这一物理现象的直接映射。它把硬件加法器的进位传播行为直接暴露为可编程的逻辑。

传承

Henry Warren 在《Hacker's Delight》中详细记载了此算法。现代 CPU 提供了 BLSI（提取最低设定位）等指令，本质上就是把 Gosper 的洞察固化到硅片里。软件的极致思考，最终变成了硬件的标配。

🔧 为什么非得用这种方式？

核心在于补码的代数封闭性。补码不只是"负数的编码方式"，它是一个完整的模运算群。x & (-x) 能work，是因为 -x = ~x + 1 中的 +1 会引发从最低位开始的进位链，正好在第一个 1 处停下。这是进位链物理行为的数学映射，不是巧合。

⚡ 替代方案？

可以用查表法（预生成所有同 popcount 的数排序后做二分查找），但内存开销爆炸；可以用循环逐个试探 while(popcount(++x) != k)，但这可能需要扫描数百万个无效数。Gosper 的魔力在于：它把一个"搜索问题"变成了纯粹的"代数计算"。搜索是 O(间距)，计算是 O(1)。

🌐 可推广性

这种"从最低有效位提取特征 → 利用进位传播改变状态 → 归一化补偿"的三段式范式，在位运算领域反复出现。如 SNOOB（Same Number of One Bits）、子集遍历 s = (s - x) & x 等。本质是同一种思维：把加法器的进位链当作免费的"搜索引擎"来使用。

💡 启示

这段代码写于 1972 年。半个世纪后的 CPU 架构师们把 BLSI、BLSR、BLSMSK 做成了硬件指令（BMI1 指令集）。这说明什么？说明真正深刻的算法洞察不会被时代淘汰 —— 它会下沉到硬件层，从"聪明的软件技巧"变成"理所应当的物理电路"。这才是 hack 的最高境界：黑进硅片里。

📋 完整代码 

unsigned gosper_next(unsigned x) {
  unsigned c = x & (-x);       // 最低位1
  unsigned r = x + c;          // 进位传播
  unsigned ones = ((r^x)/c)>>2; // 补回1
  return r | ones;             // 合并
}
O(1) 时间 O(1) 空间 无分支 位运算
