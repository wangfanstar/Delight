CHAPTER 03 · POWER-OF-2 BOUNDARIES

2 的幂边界

// 对齐、取整、边界检测——全都藏在位运算里

内存按2的幂分块是整个计算机体系结构最底层的物理事实——缓存行64字节，页面4KB，总线宽度64位。 这一章教你用位运算，把所有跟"对齐到2的幂"相关的问题，变成1~5条指令。 没有循环，没有除法，没有分支——只有掩码、右移和加一。

对齐取整 flp2 / clp2 边界穿越检测 内存分配器 无分支
3–1
对齐到已知2的幂的倍数
核心公式 把无符号整数 x 对齐到 8 的倍数（向下取整）：x & −8。 向上取整：(x + 7) & −8。 掌握了这两个，任何2的幂对齐都是同样套路，只换数字。

C — 编译期已知对齐量
// 向下：清除低k位 round_down(x) = x & -8 // 8=2^3 // 向上：先+7再清低位 round_up(x) = (x + 7) & -8 // 补齐量 padding(x) = (-x) & 7
C — 运行时对齐量 k = log₂(align)
unsigned align_down(unsigned x, int k) { return x & (~0u << k); } unsigned align_up(unsigned x, int k) { unsigned mask = (1u << k) - 1; return (x + mask) & ~mask; }
⚠ 有符号整数陷阱 "向下" 对有符号数意味着向负无穷方向取整：(-37) & (-8) = -40，不是 -32。 这在堆栈地址对齐时完全正确，但用于数组索引时要三思。
3–2
舍入到最近2的幂 — flp2 / clp2
核心思路 flp2(x) = 不超过 x 的最大2的幂；clp2(x) = 不小于 x 的最小2的幂。 关键技巧：右移传播最左侧的1，让所有低位全部变成1，再做一次"x − (x>>1)"或"+1"。 12条指令，无分支，无循环，比任何循环方案都快。

clp2 执行步骤（x−1 → 右移传播 → +1）：
flp2 — 向下取整到2的幂
unsigned flp2(unsigned x) { x = x | (x >> 1); x = x | (x >> 2); x = x | (x >> 4); x = x | (x >> 8); x = x | (x >> 16); return x - (x >> 1); // 清除最右侧以外的1 } // 或使用 nlz: 1u << (31 - nlz(x))
clp2 — 向上取整到2的幂
unsigned clp2(unsigned x) { x = x - 1; // 先减1（处理本身是2的幂） x = x | (x >> 1); x = x | (x >> 2); x = x | (x >> 4); x = x | (x >> 8); x = x | (x >> 16); return x + 1; // +1得到下一个2的幂 }
flp2 vs clp2 — 取值对照
为何先减一？ 如果 x 本身恰好是2的幂（如 x=8），不先减一的话，clp2(8) 会错误地给出 16。 先做 x−1=7，传播后全1=7，+1=8，结果正确。这是边界情况处理的经典手法： 先处理边界，再做通用流程。

检测是否穿越2的幂边界

IBM/370 汇编原理等价 C
几何直觉 a & 7 是 a 在其块内的偏移量（0到7）；8 − (a & 7) 是从 a 到块末尾还剩多少空间。 如果 l 大于这个剩余空间，必然穿越边界。这个几何解释来自 [Kumar]，是本章最美的图示。
APP
真实应用场景
场景 01
🔧 内存分配器
size = clp2(requested)
jemalloc、tcmalloc 等 slab 分配器把请求的字节数舍入到下一个2的幂 size class， 再从对应的 free list 里取块。clp2 把这个"找合适桶"的操作压缩成12条无分支指令。
场景 02
📄 DMA 页面对齐
buf = (ptr+4095) & −4096
DMA 缓冲区必须对齐到页面边界（4KB/64KB）。 malloc 返回的地址未必对齐，用 (ptr+N−1) & −N 一条指令完成对齐， 比用 posix_memalign 更轻量（已有超分配的场景下）。
场景 03
🗜 哈希表扩容
new_cap = clp2(old_cap * 2)
哈希表容量保持2的幂，使 h % capacity 变成 h & (capacity−1)—— 一条 AND 替代一次除法，快2~10×。扩容时用 clp2 确保新容量仍是2的幂。
场景 04
🌐 网络协议 MTU
frag = crosses_boundary(off, len)
IP 分片检测：给定包偏移和长度，判断是否跨越64K边界（旧式 IP 分片的限制）。 §3-3 的 crosses_boundary 直接用于内核协议栈的发包路径，5条指令无分支。
场景 05
🎮 游戏引擎纹理
tw = clp2(img_width)
OpenGL 旧版（ES 2.0以前）要求纹理宽高必须是2的幂（NPOT限制）。 引擎加载任意尺寸图片时，用 clp2 算出 POT 尺寸，再 blit + 补 padding。
场景 06
📡 SIMD 向量对齐
p = align_up(ptr, 32)
AVX 需要 32 字节对齐，AVX-512 需要 64 字节对齐。 对齐检查和对齐取整是编译器向量化 pass 和手写 SIMD kernel 的必备操作。 错误对齐会导致段错误或 10× 性能惩罚。

CRITIC
深度点评：这一章真正在说什么

本章的历史必然性

为什么是2的幂？

2的幂是计算机硬件不得不做出的妥协，不是什么优雅设计。早期内存条的地址解码器用二进制选择行列， 天然就是2的幂。缓存行大小、TLB 页表项粒度、总线宽度，全是这条规律的后果。 程序员面对的是一个"2的幂宇宙"，这不是自由选择，是物理约束。 理解这一点，就理解了为什么整个操作系统内核里到处都是 & (N-1) 和 | (~(N-1))。

右移传播：一个被低估的技巧

flp2/clp2 的核心——右移1、2、4、8、16步——表面上看是蛮力枚举， 实质上是对"二分传播"的完美利用：每次移位操作覆盖的范围翻倍，5步覆盖32位， 这是一种自描述的算法结构。很多人第一次看到这段代码会觉得它很机械、很丑， 但想想：你能找到更短、更快、更可靠的无分支方案吗？循环方案要 O(log n) 次迭代， 每次还有条件跳转；这里是严格 12 条直线指令，无分支，流水线友好。 有时候，"丑"是另一种极致的美。

IBM/370 那段代码的意味

边界穿越那道题，原书特意引用了1970年代 IBM System/370 的汇编解法 [CJS]。 这不是炫古董——它在说：好的算法思想不随硬件更迭而腐朽。 70年代的 IBM 大型机程序员用 OR+ALR 两条指令检测页面跨越， 50年后你在 ARM 服务器上写 DMA 驱动，换成 C 的位运算，逻辑完全相同。 技术会更迭，但把"进位"当信号、把"掩码"当选择器的思维方式，是永恒的。

有没有更好的实现方式？

现代 x86 有 LZCNT/BSR，ARM64 有 CLZ， 可以用 1<<(31−nlz(x)) 直接算出 flp2，只需2条指令。 但注意：这依赖硬件指令的可用性。在 FPGA、嵌入式 RISC-V、或者某些 DSP 上， 你只有移位和逻辑运算。原书的12指令方案是最大公约数——在所有平台上都有效。 好的底层工程师要同时掌握两个世界：知道最优的硬件路径，也知道最通用的软件路径。

能推广到其他技术吗？

当然。右移传播的思路直接催生了并行前缀（Parallel Prefix）运算—— 这是 GPU 计算中 scan/reduce 的核心原语，也是并行排序、并行前缀和的基础。 你在 CUDA 里写 warp-level reduce，用的是同样的"每次翻倍覆盖范围"的逻辑。 对齐取整在文件系统（块对齐）、编译器（栈帧对齐）、内核（slab 分配）里无处不在。 这一章是计算机系统底层的共同语言，不懂它，你读内核代码、读编译器代码都会遇到障碍。

优缺点的坦诚评价

优点：无分支、流水线友好、可移植、常量时间、零依赖。 在性能关键路径上（分配器、内核中断处理、协议栈快速路径）这些特性价值极大。
缺点：可读性差——任何维护者第一次看到 x = x | (x >> 16) 都不会直觉上明白它在干什么。必须加注释，必须有文档，否则这是技术债。 另外，clp2(0) = 0，clp2(x) 对 x>2³¹ 溢出返回0——调用方必须处理这两个边界。 位操作技巧不是银弹，它是特定场景下的正确工具，在合适的地方用合适的锤子。

相关技术发展时间线
1964
IBM System/360 — 页对齐成为标准
S/360 引入统一的内存寻址模型，页面大小固定为2的幂。OR+ADD 检测边界穿越的技法由此产生，一直沿用至今。
1972
HAKMEM — 并行前缀技巧文档化
MIT AI Lab 的 HAKMEM 备忘录将大量位操作技巧系统记录，其中包括右移传播和2的幂边界相关操作。
1990s
jemalloc / tcmalloc — clp2 进入工业界
现代内存分配器大量使用 clp2 进行 size class 映射。clp2 成为每次 malloc 快速路径上几乎必然出现的操作。
2003
Hacker's Delight 第一版 — 系统整理
Warren 将分散在各处的位操作智慧系统整理成书，Ch.03 将2的幂边界处理从"老程序员的经验"变成可传授的知识体系。
2012
x86 BMI1/LZCNT — 硬件加速
Intel Haswell 引入 LZCNT 指令，把 flp2 从12条指令压缩到2条。但软件方案在嵌入式和跨平台场景仍是首选。
2021
C++20 <bit> — 标准化
std::bit_floor() 和 std::bit_ceil() 正式进入 C++ 标准，是 flp2/clp2 的标准化版本。从此可以无顾虑地用标准库——编译器会自动选择最佳指令。
C++20 标准化版本 vs 位操作手写
C++20 <bit> 标准库

// 跨平台，无硬件依赖 unsigned clp2(unsigned x) { x--; x |= x >> 1; x |= x >> 2; x |= x >> 4; x |= x >> 8; x |= x >> 16; return ++x; } // RISC-V, FPGA, 老 DSP 唯一选择
