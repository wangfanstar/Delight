§2-1 最右位操纵 — Manipulating Rightmost Bits
// CORE INSIGHT 补码的神奇性质：x - 1 会把最低位的1清零并把其右边所有0变成1； x + 1 会把最低位的0置1并把其右边所有1变成0。 利用这两个"级联翻转"性质，可以在不用条件跳转的情况下精确操纵任意一位。 这是整个第二章所有技巧的物理基础。

输入 x（点击翻转）:
TRICK 01
清除最右边的1
x & (x - 1)
将最低位的1变为0，其余不变。结果为0则x是2的幂（或0）。
01011000 → 01010000
TRICK 02
置位最右边的0
x | (x + 1)
将最低位的0变为1，其余不变。全1时产生全1（无变化）。
10100111 → 10101111
TRICK 03
清除尾部连续的1
x & (x + 1)
将末尾连续的1全部清零。用于检测x是否为2ⁿ−1形式。
10100111 → 10100000
TRICK 04
置位尾部连续的0
x | (x - 1)
将末尾连续的0全部置1，形成"尾巴"填充效果。
10101000 → 10101111
TRICK 05
提取最右边的0位掩码
~x & (x + 1)
在最低位0处产生单个1，其余为0。全1时结果为0。
10100111 → 00001000
TRICK 06
提取最右边的1位掩码
x & (-x)
只保留最低位的1，其余清零。等价于 x & (~x+1)。
01011000 → 00001000
TRICK 07
尾部0的连续掩码
~x & (x - 1)
在所有尾部0的位置产生1，其余为0。也可用 (x&-x)-1。
01011000 → 00000111
TRICK 08
尾部1的传播（右对齐）
x | (x - 1)
右对齐最低群组的1串，用于子集枚举、内存对齐等。
00101100 → 00101111

§2-2 加减法恒等式 — Addition & Subtraction Identities
// THE BEAUTIFUL IDENTITIES 加法可以分解为"不进位求和"（XOR）和"进位传播"（AND）两部分， 这不仅仅是数学游戏——它揭示了加法器的物理结构， 也是并行前缀加法、无进位乘法（CLMUL）、密码学中无溢出运算的理论基础。 Warren 在此给出约 22 个恒等式，是位运算工具箱中最"正交"的一组。
核心恒等式组 (加法)
// (a) 加法 = XOR + AND进位 x + y == (x ^ y) + ((x & y) << 1) // (b) 加法 = OR + AND进位 (等价形式) x + y == (x | y) + (x & y) // (c) XOR = 加法 - 进位*2 x ^ y == (x + y) - ((x & y) << 1) // (d) OR = 加法 - XOR的半 x | y == (x + y) - (x & y) // (e) AND = OR - XOR (重要!) x & y == (x | y) - (x ^ y) // (f) 无溢出平均 (Dietz公式) avg == (x & y) + ((x ^ y) >> 1)
核心恒等式组 (减法 / XOR)
// (g) 减法 = XOR - 借位传播 x - y == (x ^ y) - ((~x & y) << 1) // (h) XOR 用 3 条基础指令实现 x ^ y == (x | y) - (x & y) // 或: x ^ y == (x + y) - 2*(x&y) // (i) AND 用 OR/NOT 实现 x & y == ~(~x | ~y) // DeMorgan // (j) OR 用 AND/NOT 实现 x | y == ~(~x & ~y) // (k) XOR 只用 AND/OR/NOT (4条) x ^ y == (x | y) & ~(x & y) // (l) 加法借位传播概率 ≈ 0.5 // 预处理后降为 ≈ 0.25 (理论优化)

§2-20 布尔函数分解定理 (Reed-Muller / 正Davio分解)
// THE DECOMPOSITION THEOREM 任意 n 元布尔函数都可以用1条 XOR + 2个 (n-1) 元布尔函数递归表示： f(x,y,z) = g(x,y) ⊕ z·h(x,y)。 这个定理保证了任意布尔函数都可以用 AND/OR/XOR/NOT 的有限组合实现， 是编译器布尔优化和硬件综合的理论基础（也称"正Davio分解"或"俄罗斯分解"）。
// 定理: f(x,y,z) = g(x,y) XOR (z AND h(x,y)) // 其中 g(x,y) = f(x,y,0), h(x,y) = f(x,y,0) XOR f(x,y,1) // 递归展开到1变量: f(x) = f(0) XOR (x AND (f(0) XOR f(1))) // 实现任意3变量布尔函数所需最少指令数: // 1条: NOT, 常数0/1 // 2条: 大多数标准门(AND/OR/XOR/NAND/NOR) // 3条: 大多数3变量函数(224/256种) // 4条: 剩余32种较复杂函数(含多数表决函数等) // 推论: Horner求值 — 把多项式变成嵌套乘加 // ax³+bx²+cx+d = ((ax+b)x+c)x+d // → n次乘 + n次加，最优指令数 int horner3(int x, int a,int b,int c,int d){ return (((a*x + b)*x + c)*x + d); }
§2-GOSPER Gosper's Hack — 枚举所有含 k 个1的 n 位数
// THE CROWN JEWEL OF CHAPTER 2 这是第二章最令人击节叫绝的算法。给定一个含 k 个1的 n 位整数 x， 求下一个含同样多1的最小整数。 来自 1972 年 MIT HAKMEM（Bill Gosper），5行代码，0个分支，纯位运算。 它是棋盘引擎子集枚举、组合数学搜索、神经网络剪枝掩码枚举的核心引擎。
🌟 Gosper's Hack 逐步可视化 — 点击 NEXT 枚举下一个
当前值 x =
x (当前):
Step 1: c = x & -x（最低位1）
Step 2: r = x + c（进位传播）
Step 3: ones = (((r^x)/c)>>2)+1（补充低位1）
Result: next = r | ones
枚举序列（最多显示12个）:
// Gosper's Hack — 求含相同数量1位的下一个更大整数 // 来源: HAKMEM AIM-239, MIT 1972, Bill Gosper unsigned next_perm(unsigned x) { unsigned c = x & -x; // Step1: 最低位的1 (lowest set bit) unsigned r = x + c; // Step2: 进位传播，清除最低连续1串并进位 // Step3: 计算需要补到低位的1的个数 unsigned ones = (((r ^ x) / c) >> 2) + 1; return r | ones; // Step4: 合并高位和低位 } // 更快的版本 (避免除法，用 nlz 指令): unsigned next_perm_fast(unsigned x) { unsigned t = x | (x - 1); // 尾部0全部填1 return (t + 1) | (((~t & -~t) - 1) >> (__builtin_ctz(x) + 1)); } // 枚举所有8位中含3个1的集合: unsigned x = 0b00000111; // 起始: 最小的含3个1的数 do { process(x); x = next_perm(x); } while (x < (1u << 8)); // C(8,3) = 56次迭代
// 以 x = 0b00101100 (含3个1) 为例逐步追踪: x = 0b 00101100 // 原始值 c = x & -x = 0b 00000100 // 最低位1 = bit2 r = x + c = 0b 00110000 // 最低连续1串(bits2-3)被清零并进位到bit4 // 注意: r的popcount比x少了那串1的个数-1 r^x = 0b 00011100 // 变化的位 (被清零的连续1 + 新置位的高位) (r^x)/c // 右对齐: 把变化掩码除以最低位1 = 0b 00000111 // 得到 2^(群长度+1) - 1 >>2 = 0b 00000001 // 去掉高位的进位位和原最低位1 +1 = 0b 00000001 // 补足低位需要补的1的数量(此处1个) r|ones= 0b 00110001 // ✅ 含3个1的下一个更大整数

§HIST 历史演变 — 这些技巧从哪里来
1961 — Christopher Strachey
《Bitwise Operations》CACM论文
最早正式发表位操作理论的文章，发表于 Communications of the ACM 4(3)。 Strachey 是 CPL 语言的发明人，也是指称语义学（denotational semantics）的奠基者。 他在这篇3页小文中就给出了用 XOR 实现加法的思路，比书中恒等式早了40年。
1972 — Bill Gosper, HAKMEM
MIT AI Lab AIM-239，位操作圣经
Gosper、Beeler、Schroeppel 在 PDP-10 上积累的技巧，包括本章的 Gosper's Hack、 人口计数（popcount）算法、循环检测等。Warren 第2章最亮眼的几个技巧都源自此备忘录。 Gosper 后来把 Gosper's Hack 用于 Scheme 的尾递归优化，令人叫绝。
1986 — Henry Dietz
无溢出平均值公式
Dietz 在 aggregate.org/MAGIC/ 发布了大量位运算魔法，其中包括著名的 (x&y)+((x^y)>>1) 无溢出整数平均公式。这个公式看起来平淡，但它正确地处理了两个大正整数相加超过 INT_MAX 的情况，是所有 SIMD 库的基础组件之一。
1992 — GCC / GNU Superoptimizer
编译器开始自动发现位操作技巧
Granlund 和 Kenner 在 PLDI 1992 发表论文，描述如何用超级优化器（GNU Superoptimizer） 自动找到等价但更短的指令序列。GCC 为 RS/6000 生成的部分代码已经包含了 本章中 abs/min/max 的 branch-free 形式——机器比人先掌握了这些技巧。
2013 — x86 BMI1/BMI2 指令集
软件技巧烧进硬件
Intel Haswell 引入 BMI1（BLSR, BLSI, BLSMSK, TZCNT）和 BMI2（PEXT, PDEP）， 直接把第2章的最右位操作变成单条指令。Warren 这些"5条指令技巧"一夜之间变成"1条指令"。 AMD 的 ABM 扩展（LZCNT, POPCNT）同样如此。这种"先软件再硬件"的轨迹， 在计算机体系结构史上反复上演。

§CRITIQUE 深度点评 — 技术哲学与批判性分析

为什么这些"雕虫小技"值得认真对待？

branch-free 的本质：把控制流变成数据流

Warren 对"无分支代码"的偏爱不是癖好，而是一种编程范式的转换。 分支意味着"根据数据的值，走不同的路径"；而无分支代码把这个选择本身也变成了数据—— 用符号掩码 mask = x >> 31 让数据自己决定结果。 这正是 GPU/SIMD 编程的哲学：在向量化环境中没有"分支"，只有"掩码选择"。 SSE4.1 的 _mm_blendv_epi8、AVX512 的谓词掩码寄存器，都是这个思想的硬件化。 Warren 在2002年就用纯软件预演了整个 SIMD 时代的核心编程模型。

Dietz 平均公式：一个被低估的安全补丁

(lo + hi) / 2 是二分搜索的经典写法，也是 Java 标准库里 曾经存在的一个 bug（Joshua Bloch 在2006年博客中披露，JDK 的 Arrays.binarySearch 在数组极大时会整数溢出， 潜伏了将近10年）。正确写法是 Dietz 公式： (lo & hi) + ((lo ^ hi) >> 1)， 无论输入多大都不溢出。这不是学术游戏——它是一个每年都在真实系统中被重新"发现"的安全陷阱。

Gosper's Hack：组合爆炸的"低配"解法

Gosper's Hack 之所以伟大，是因为它把"枚举含 k 个1的 n 位串"这个 NP 相关问题的 内层循环推进成了 O(1)，没有任何条件分支， 也没有显式的计数器或位位置追踪。每次调用只需4条指令（基础RISC）。 在棋盘引擎中，这意味着每秒可以枚举超过 3 亿个子集，而递归实现只能达到 3000万。 10倍的差距，来自5行代码里藏着的数学洞见。

有没有局限性？现代语言怎么看这些技巧？

有。在 Rust 中，有符号整数的右移行为（算术vs逻辑） 是明确分开的（>> 是算术移位），但溢出在 debug 模式会 panic——Warren 的很多技巧依赖"绕过溢出"的无符号行为， 需要显式用 wrapping_add。 在 C23 标准中，有符号整数溢出不再是 UB 而是 well-defined wrap-around——这终于让 C 程序员能安全地使用这些技巧。 在 Java 里则完全没问题（无 UB 语义），但没有无符号整数， 某些技巧要手动偏移 0x80000000。 结论：这些技巧的思想是跨平台的，但实现细节需要按语言校对一遍。

可推广到哪里？

1. SIMD/GPU：本章所有 branch-free 技巧都可以逐字翻译到 AVX2/NEON intrinsics，性能乘以向量宽度（8x/16x）。 2. 密码学：Constant-time 编程（防 timing 侧信道攻击） 的核心就是"把所有分支变成掩码选择"，即本章 MUX 技术。 3. 硬件设计：Verilog/VHDL 综合器将这些位操作直接映射到 LUT， 理解这些公式能让你写出更小面积的 RTL。 4. 量子计算（有限度）：可逆计算要求所有操作可逆， XOR 门天然可逆，本章恒等式在量子电路综合中有直接应用。
