➕ 加减法恒等式 - Addition & Subtraction Identities
💡 核心洞察： 加法可以分解为两个独立部分：不进位求和(XOR) 和 进位传播(AND左移)。这不仅是数学游戏——它揭示了加法器的物理结构， 也是并行前缀加法、无进位乘法(CLMUL)、密码学的理论基础。
公式编号 恒等式 含义与应用
(a) x + y == (x^y) + ((x&y)<<1) 加法 = XOR(无进位) + AND(进位)*2
王牌恒等式
(b) x + y == (x|y) + (x&y) 加法 = OR + AND（等价分解）
(c) x ^ y == (x+y) - ((x&y)<<1) 反向推导：XOR = 加法 - 进位*2
(d) x | y == (x+y) - (x&y) OR 用加减表示
(e) ⭐ x & y == (x|y) - (x^y) AND 用 OR/XOR 表示
关键的对偶性
(f) ⭐⭐ avg = (x&y) + ((x^y)>>1) Dietz 无溢出平均
永远不会整数溢出！
(g) x ^ y == (x|y) & ~(x&y) XOR = (x|y) & NOT(x&y)
4 个操作的分解
🧠 De Morgan 对偶律
• x & y == ~(~x | ~y) ← AND/OR互换 + NOT
• x | y == ~(~x & ~y)
• x ^ y == (x|y) & ~(x&y) ← 最直观的 XOR 分解
• 记忆诀窍：AND/OR/XOR 中任意两个可以推导第三个
🎮 恒等式验证器 - 自动计算检验所有公式
🎓 理论意义
这 22 个恒等式（Warren 原著列出）是位运算的"正交基"。任意两个基础操作(AND/OR/XOR/NOT)的组合都能用其他表示。 这说明在信息论层面，位运算的表达能力是等价的，只是指令数成本不同。

⚡ 无分支算术 - Branch-Free Magic (abs/min/max/avg/sign)
💡 关键哲学： 把条件判断转化为纯位操作。用 符号掩码 mask = x >> 31 （正数是全 0，负数是全 1）让数据自己决定结果。这正是 SIMD/GPU 编程的核心思想！
| 🔑 符号掩码
mask = x >> 31
✅ x ≥ 0: mask = 0x00000000 (全 0)
❌ x < 0: mask = 0xFFFFFFFF (全 1)
基础工具
📊 A 绝对值 abs(x)
abs = (x + mask) ^ mask
3 条指令，0 个分支
等价于：x < 0 ? -x : x
优化版本：(x ^ mask) - mask
高频使用
📉 B 最小值 min(x,y)
min = y + ((x-y) & ((x-y)>>31))
3 条指令，0 个分支
等价于：x < y ? x : y
原理：差值的符号位决定选择
📈 C 最大值 max(x,y)
max = x - ((x-y) & ((x-y)>>31))
3 条指令，0 个分支
等价于：x > y ? x : y
⚖️ D 平均值 avg(x,y)
avg = (x & y) + ((x ^ y) >> 1)
Dietz 公式
无论多大都不会溢出！
比 (x+y)/2 安全 100 倍
★★★ 关键
➕ E 符号函数 sign(x)
sign = (x >> 31) | (-x >> 31)
结果：-1 (负) / 0 (零) / +1 (正)
2 条右移 + 1 条或 = 3 条指令
❌ 传统方式 (有分支)
int abs(int x) { if (x < 0) return -x; return x; }
✅ 无分支方式
int abs(int x) { int m = x >> 31; return (x + m) ^ m; }
🎮 无分支操作演示 - 改变 x、y 实时计算
🧠 记忆诀窍
• mask = x >> 31 是"开关"，正数 off，负数 on
• abs：若 x<0 则加掩码使其反号，然后异或回来
• min/max：用差值的符号位决定加还是减
• avg：共同部分 AND，差异部分右移 1 位
🏆 性能对比
传统 abs()：分支预测错误时 10~20 个周期
无分支版本：恒定 3~4 个周期，且能完全向量化（SIMD）
在紧循环中，无分支代码快 5~10 倍！这就是 Warren 思想的商业价值。

🔀 MUX 多路选择器 - Multiplexer
💡 概念： MUX(x, y, mask) 用掩码在 x 和 y 之间做条件选择。这是 SIMD 中 blendv 指令的原型，也是常量时间密码学的核心工具。
🔀 标准 MUX - 标准形式
mux(x,y,m) = (x&m) | (y&~m)
m = 全 1: 选 x
m = 全 0: 选 y
等价于三元运算符 m ? x : y
4 条指令
⚡ 优化 MUX - 优化形式
mux(x,y,m) = ((x^y)&m) ^ y
少 1 条指令（只需 3 条）！
需要处理器支持 ANDC
(AND-NOT-COMPLEMENT) 指令
ARM NEON、x86 AVX512 都支持
高级优化
🧠 MUX 应用场景
• SIMD：SSE4.1 _mm_blendv_epi8 / ARM NEON vbslq_u32
• 密码学：Constant-time 编程防 timing 侧信道
• CPU 微码：条件传送指令 CMOV 的原型
• 无分支 abs：t = x >> 31; abs = mux(-x, x, t)

🌟 Gosper's Hack - 枚举组合的王牌
💡 王冠宝石 👑
给定一个含 k 个 1 的 n 位整数 x，求下一个含同样多 1 的最小整数。 来自 1972 年 MIT HAKMEM（Bill Gosper），5 行代码，0 个分支，纯位运算。 这是棋盘引擎子集枚举、组合搜索、稀疏向量优化的核心引擎。

🎁 特殊技巧合集
⚠️ 一 有符号加法溢出检测
ov = ((x^sum) & (y^sum)) >> 31
检测 x + y 是否溢出
ov = -1 (0xFFFFFFFF): 溢出
ov = 0: 不溢出
原理：同号相加得反号 = 溢出
🔒 二 饱和加法 (Saturating)
溢出时钳制到 MAX/MIN
用途：音频、图像处理防截断
硬件：ARM QADD, x86 _mm_adds_epi16
🔌 三 符号扩展
sign_ext(x,n) = (x<<(32-n))>>(32-n)
把 n 位有符号数扩展为 32 位
2 条指令（移位）
📍 四 位域提取/插入
提取：(x>>p) & ((1<<n)-1)
插入：(x & ~mask) | ((v<<p) & mask)
🧮 五 Horner 规则
ax³+bx²+cx+d = ((ax+b)x+c)x+d
用途：哈希函数、多项式求值
hash *= 31 ≡ (hash<<5) - hash
🌊 六 位域复制
从位置 i 复制值到位置 j
用途：位图操作、纹理坐标变换
复制 bit i 到 bit j:
x = (x & ~(1<<j)) | (((x>>i)&1)<<j)
🌊 七 多字节 SIMD 加法
四字节并行加法（4×byte packed）
用途：图像处理、向量运算
Step1: 屏蔽高位防进位穿透
lo = (x & 0x7F7F7F7F) + (y & 0x7F7F7F7F)
Step2: 修复高位
hi = (x ^ y) & 0x80808080
sum = lo ^ hi
🔢 八 双字移位 (Shld/Shrd)
64 位值用两个 32 位寄存器表示
指令：x86 SHLD/SHRD
hi = (hi << n) | (lo >> (32-n))
lo = lo << n
⚠️ n=0 或 32 需特殊处理

🎓 布尔分解定理 - Reed-Muller / Davio 分解
💡 数学原理： 任意 n 元布尔函数都可以用一条 XOR + 两个 (n-1) 元布尔函数递归表示：

f(x,y,z) = g(x,y) ⊕ z·h(x,y)

其中 g(x,y) = f(x,y,0)，h(x,y) = f(x,y,0) ⊕ f(x,y,1)

这个定理保证了任意布尔函数都可以用 AND/OR/XOR/NOT 的有限组合实现， 是编译器布尔优化和硬件综合的理论基础。
📊 实现复杂度
• 1 条指令：NOT、常数 0/1
• 2 条指令：大多数标准门 (AND/OR/XOR/NAND/NOR)
• 3 条指令：大多数 3 变量函数 (224/256 种)
• 4 条指令：剩余 32 种较复杂函数（含多数表决 MAJ 等）
🧠 Horner 规则扩展
多项式求值：ax³+bx²+cx+d = ((ax+b)x+c)x+d
• n 个变量 = n 次乘法 + n 次加法（最优）
• Java String.hashCode() 就用的这个
• 乘以 31 的快速写法：hash * 31 = (hash << 5) - hash

⏰ 历史演变 - 这些技巧从哪里来
1961 — Christopher Strachey
《Bitwise Operations》CACM 论文
最早正式发表位操作理论的文章。Strachey 是 CPL 语言发明人、指称语义学奠基者。 他在 3 页小文中就给出了用 XOR 实现加法的思路，比 Warren 恒等式早了 40 年。
1972 — Bill Gosper, HAKMEM
MIT AI Lab AIM-239，位操作圣经
Gosper、Beeler、Schroeppel 在 PDP-10 上积累的技巧。包括本章最亮眼的 Gosper's Hack、 人口计数、循环检测等。Warren 第 2 章最经典的几个技巧都源自此备忘录。
1986 — Henry Dietz
无溢出平均值公式
Dietz 在 aggregate.org 发布了大量位运算魔法。著名的 (x&y)+((x^y)>>1) 无溢出平均公式看起来平淡，但它正确处理了两个大正整数相加超过 INT_MAX 的情况， 是所有 SIMD 库的基础组件。
1992 — GCC / GNU Superoptimizer
编译器开始自动发现位操作技巧
Granlund 和 Kenner 在 PLDI 1992 发表论文，描述如何用超级优化器自动找到等价但更短的指令序列。 机器比人先掌握了这些技巧！
2006 — Joshua Bloch, Java Bug Disclosure
二分搜索溢出 bug
JDK 的 Arrays.binarySearch 在数组极大时会整数溢出。 正确写法是 Dietz 公式 (lo & hi) + ((lo ^ hi) >> 1)。 这个陷阱潜伏了 近 10 年，每年都被重新"发现"。
2013 — x86 BMI1/BMI2 指令集
软件技巧烧进硬件
Intel Haswell 引入 BMI1（BLSR, BLSI, BLSMSK, TZCNT）和 BMI2（PEXT, PDEP）， 直接把第 2 章的最右位操作变成单条指令。 Warren 的"5条指令技巧"一夜之间变成"1条指令"！
🔍 技术洞见
这种"先软件再硬件"的轨迹在计算机体系结构史上反复上演：
1️⃣ 软件工程师发明了非常规的优化技巧
2️⃣ 多年后 CPU 设计师把最受欢迎的技巧烧进硬件指令
3️⃣ 新代码用硬件指令重写，性能暴增
这正是 x86 BMI1、AVX-512、ARM NEON 的由来！

🔬 深度点评 - 技术哲学与批判分析
⚙️ 一 为什么无分支代码这么重要？
分支的代价： 现代 CPU 分支预测失败时要清空流水线，损失 10~20 个周期。 无分支代码用符号掩码 mask = x >> 31 让数据自己决定结果，完全避免分支。

SIMD 哲学： 在向量化环境中没有"分支"，只有"掩码选择"。 SSE4.1 的 _mm_blendv_epi8、AVX-512 的谓词掩码，都是这个思想的硬件化。 Warren 在 2002 年就用纯软件预演了整个 SIMD 时代的编程模型。
🛡️ 二 Dietz 平均公式：被低估的安全补丁
经典 bug： (lo + hi) / 2 是二分搜索的标准写法，但在数组极大时会整数溢出。 Joshua Bloch 在 2006 年博客披露 JDK bug，潜伏了近 10 年。

正确方案： (lo & hi) + ((lo ^ hi) >> 1) 无论输入多大都不会溢出！这不是学术游戏——它是每年都被重新"发现"的安全陷阱。
👑 三 Gosper's Hack：组合爆炸的救星
性能突破： Gosper's Hack 把"枚举含 k 个 1 的 n 位串"的内层循环推进成 O(1)， 没有条件分支，没有计数器。每次调用只需 4 条指令。

实战对比： 递归实现：~3000 万次/秒
Gosper's Hack：~3 亿次/秒
加速比：100 倍，只因为 5 行代码的数学洞见。
⚠️ 四 局限性：现代语言怎么看？
Rust 的挑战： 有符号整数溢出在 debug 模式会 panic。 需要显式用 wrapping_add 避免 UB。

C23 改进： 有符号整数溢出不再是 UB，而是 well-defined wrap-around。

Java 安全性： 无 UB 语义，完全没问题。但没有无符号整数，某些技巧要手动偏移。

结论：思想跨平台，但实现细节需按语言校对。

Q3 ★ 推导 Dietz 无溢出平均值
问题：证明
(x&y) + ((x^y)>>1)
永远不会溢出，且等于 (x+y)/2（舍入）
提示：
• x+y = (x&y)*2 + (x^y)
• (x&y)*2 + (x^y) = (x&y)*2 + ((x^y)>>1)*2
• 所以 (x+y)/2 = (x&y) + ((x^y)>>1)
Q8 ★★ MUX 3 条指令实现
问题：证明 MUX 可以用 3 条基础 RISC 指令实现
mux(x,y,m) = ((x^y) & m) ^ y
标准形式：
mux = (x&m) | (y&~m) (4 条指令)
优化形式：
mux = ((x^y)&m) ^ y (3 条指令)
Q10 ★★ 位复制问题
问题：给定 x 和两个位置 i、j， 把 x 在位 i 的值复制到位 j（其余位不变）
方案：
提取 bit i: (x >> i) & 1
清除 bit j: x & ~(1 << j)
合并: (x & ~(1<<j)) | (((x>>i)&1)<<j)
Q11 ★★★ 布尔分解定理
问题：实现任意 n 元布尔函数最少需要几条二元指令？
答案：
• 1 变量: 1 条 (NOT)
• 2 变量: 2 条 (AND/OR/XOR)
• 3 变量: 平均 3 条, 最多 4 条
• n 变量: ≈ 3n 条（Reed-Muller 展开）

Q4 ★★ 四数无溢出平均
问题：给出四个无符号整数 (a+b+c+d)/4 的无溢出计算方法
方案：
分组计算：
avg1 = (a&b) + ((a^b)>>1)
avg2 = (c&d) + ((c^d)>>1)
avg4 = (avg1&avg2) + ((avg1^avg2)>>1)
