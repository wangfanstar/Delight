📐 第一部分：§3-1 向已知2次幂对齐 (6个)
ALIGN 01

向下对齐到已知2次幂 d⭐⭐⭐

x & (-d)
d 必须是2的幂
点击查看答案
向下取整到d的倍数
x & (-d) 等价于 x & ~(d-1)

原理：-d 在二进制中恰好是低 log₂d 位全0、高位全1 的掩码

例: d=4096(0x1000), -d=0xFFFFF000
x & 0xFFFFF000 → 清零低12位 → 页对齐
ALIGN 02

向上对齐到已知2次幂 d⭐⭐⭐

(x + d - 1) & (-d)
先"顶上去"再向下对齐
点击查看答案
向上取整到d的倍数
等价写法: (x + (d-1)) & ~(d-1)

Linux内核: PAGE_ALIGN(addr)
(addr + 4095) & ~4095

原理: +d-1 保证不足一页的部分"进位"，再AND截断低位
ALIGN 03

取模/偏移量计算

x & (d - 1)
等价于 x mod d（d为2次幂）
点击查看答案
页内偏移 / 环形缓冲索引
x & (d-1) = x 除以 d 的余数

Ring Buffer 索引: idx & (size-1)
代替 idx % size，快 10~20 倍

Linux kfifo / Disruptor 核心原理
ALIGN 04

对齐检测

(x & (d-1)) == 0
检测 x 是否已对齐到 d
点击查看答案
判断是否为 d 的倍数
低 log₂d 位全为0 → 已对齐

判断2次幂: (x & (x-1)) == 0
注意: 0 也会通过判断！需要 x > 0

SIMD断言: assert((addr & 15) == 0)
ALIGN 05

-d 掩码原理 ⭐⭐

-d = ~(d-1)
二补码取负 = 高位全1掩码
点击查看答案
二补码与掩码的统一
d=4 → d-1=3=011
~(d-1)=...11100 = -4

AND掩码 = 取负操作，这是二补码的数学必然

所以 x & (-d) ≡ x & ~(d-1)
ALIGN 06

64位对齐扩展 ⭐

(x + d - 1) & ~(d - 1)
32位和64位写法完全相同
点击查看答案
跨平台对齐写法
uintptr_t align_up(uintptr_t x, size_t d)
{ return (x+d-1) & ~(d-1); }

用 uintptr_t 保证32/64位都正确
d 必须是2的幂，否则公式失效
⬇️ 第二部分：§3-2 flp2 向下取整到2次幂 (6个)
FLP2 01

flp2 定义⭐⭐⭐

flp2(x) = 2⌊log₂x⌋
不超过 x 的最大2次幂
点击查看答案
floor-power-of-2
flp2(0) = 0, flp2(1) = 1
flp2(5) = 4, flp2(8) = 8
flp2(2³¹-1) = 2³⁰

flp2(x)=clp2(x) 当且仅当 x 是2次幂或0
FLP2 02

5步 OR-Shift 位传播⭐⭐⭐

x|=x>>1; x|=x>>2;%%BR%%x|=x>>4; x|=x>>8; x|=x>>16
传播最高1-bit向右覆盖所有低位
点击查看答案
并行前缀 OR 传播 (Guy Steele)
结果: 从最高1-bit到bit0全为1
例: 0x00100000 → 0x001FFFFF

倍增策略: 1→2→4→8→16，log₂32=5步
共 5次OR + 5次SHIFT = 10条指令
FLP2 03

flp2 完整代码⭐⭐⭐

传播后: x - (x >> 1)
减去自身右移一位 = 保留最高位
点击查看答案
flp2 完整实现 (12条指令)
x|=x>>1; x|=x>>2; x|=x>>4;
x|=x>>8; x|=x>>16;
return x - (x>>1);

x-(x>>1): 0x001FFFFF→0x00100000
掩码减去半掩码 = 最高位单独1
FLP2 04

nlz 方法 (硬件最优)⭐⭐

0x80000000 >> nlz(x)
nlz = Number of Leading Zeros
点击查看答案
单指令实现 (现代 CPU)
GCC: 1u << (31 - __builtin_clz(x))
x86: BSR/LZCNT 指令
ARM: CLZ 指令

注意: x=0 时 nlz(x)=32，需特判
C++20: std::bit_floor(x)
FLP2 05

flp2 互转关系 ⭐⭐

clp2(x) = flp2(2x - 1)
x > 0
用 flp2 算 clp2
点击查看答案
flp2 与 clp2 互推
clp2(x) = flp2(2x-1) for x > 0
flp2(x) = clp2(x/2+1) for x > 1

性质: flp2(2ⁿx) = 2ⁿ·flp2(x)
对偶: clp2(x) = 1/flp2(1/x), x≠0
FLP2 06

64位扩展 ⭐⭐

追加: x |= x >> 32
32位→64位只需多一步
点击查看答案
64位 flp2 (6步 OR-Shift)
x|=x>>1; x|=x>>2; x|=x>>4;
x|=x>>8; x|=x>>16; x|=x>>32;
return x - (x>>1);

log₂64=6步，规律: 步数=log₂(位宽)
⬆️ 第三部分：§3-2 clp2 向上取整到2次幂 (6个)
CLP2 01

clp2 定义⭐⭐⭐

clp2(x) = 2⌈log₂x⌉
不小于 x 的最小2次幂
点击查看答案
ceiling-power-of-2
clp2(0) = 0, clp2(1) = 1
clp2(5) = 8, clp2(8) = 8
clp2(2³¹+1) = 0 (溢出!)

HashMap/FFT/RingBuffer 的标配操作
CLP2 02

减1的妙手⭐⭐⭐

x -= 1 ← 灵魂！
让2次幂"原地保持"不变
点击查看答案
为什么先减1？
x=8(1000): 不减1→传播→1111→+1→16 ❌
x=8(1000): 减1→7(0111)→传播→0111→+1→8 ✅

区分"已在边界"和"在边界之间"的关键
CLP2 03

clp2 完整代码⭐⭐⭐

x--; 传播5步; x++;
减1 → OR-Shift → 加1
点击查看答案
clp2 完整实现 (13条指令)
x -= 1;
x|=x>>1; x|=x>>2; x|=x>>4;
x|=x>>8; x|=x>>16;
return x + 1;

比 flp2 多 1 条指令 (sub + add)
CLP2 04

nlz 方法 ⭐⭐

1 << (32 - nlz(x-1))
硬件 CLZ 指令一步到位
点击查看答案
现代 CPU 单指令实现
1u << (32 - __builtin_clz(x-1))

Java: Integer.highestOneBit(x-1) << 1
Java HashMap: tableSizeFor(cap)
C++20: std::bit_ceil(x)

x=0 或 x=1 需特判!
CLP2 05

溢出陷阱 ⚠️⭐⭐⭐

clp2(2³¹+1) = 0 !
32位无符号无法表示 2³²
点击查看答案
⚠️ 必须检查溢出
result = clp2(x);
if (result < x) // 溢出了！

Java HashMap 的 tableSizeFor 有此检查
malloc(clp2(x)) 若 x>2³¹ 则 malloc(0) → 灾难
CLP2 06

Java tableSizeFor ⭐⭐

n=cap-1; n|=n>>>1;
...n>>>16; n+1
JDK HashMap 源码 = Warren clp2
点击查看答案
Josh Bloch 用了 Warren 的算法
完全就是 clp2(cap)！
int n = cap - 1;
n |= n >>> 1; ... n |= n >>> 16;
return (n < 0) ? 1 : n + 1;

Java 用 >>> (无符号右移) 代替 >>
🔍 第四部分：§3-3 2次幂边界跨越检测 (6个)
CROSS 01

XOR 跨越检测⭐⭐⭐

((a ^ (a+l-1)) >> k) != 0
起始和结束地址 XOR 后看高位
点击查看答案
是否跨越 2ᵏ 边界
从地址 a 传输 l 字节，是否跨越 2ᵏ 边界？

XOR 在不同位置显示差异：高于第k位不同 → 跨越
l ≥ 1，l=0 需特判（会下溢）
CROSS 02

4KB页跨越检测⭐⭐⭐

((a ^ (a+l-1)) >> 12) != 0
4KB = 2¹²，k = 12
点击查看答案
DMA 传输页面跨越检测
等价写法：(a & 0xFFF) + l - 1 >= 4096
即：页内偏移 + 长度 - 1 ≥ 页大小

DMA 驱动必备：跨页需拆成两次传输
虚拟内存映射、IOMMU 边界检测同理
CROSS 03

IBM System/370 方案 ⭐⭐

¬(a | -d) < l
Christopher J. Stephenson [CJS]
点击查看答案
System/370 三指令经典方案
等价方案（更直观）:
(a & (d-1)) + l > d
页内偏移+长度 > 页大小 → 跨越

Warren 书中引用 [CJS] 私人通信
IBM 大型机程序员几十年前的经典技巧
CROSS 04

l=0 边界陷阱 ⚠️ ⭐⭐

l=0 时 a+l-1 下溢!
无符号减法下溢 = 假跨越
点击查看答案
⚠️ 必须前置检查 l
l=0 时 a+0-1 → 0xFFFFFFFF (下溢)
XOR检测会误报"跨越"！

if (l == 0) return NO_CROSSING;
Linux DMA 驱动中有此前置检查
CROSS 05

XOR 原理 ⭐⭐

a ⊕ (a+l-1)
高于第k位不同 → 跨越
XOR 是最优差异检测器
点击查看答案
XOR 差异检测原理
同一2ᵏ块内: XOR >> k = 0 (高位相同)
跨越2ᵏ块: XOR >> k ≠ 0 (高位不同)

类比: 同一楼层高位相同，跨层高位不同
不需要取模！纯位操作，1~2条指令
CROSS 06

缓存行跨越 ⭐⭐

((a^(a+l-1)) >> 6) != 0
64B缓存行: k=6
点击查看答案
性能优化：避免跨缓存行
64字节缓存行 = 2⁶，k=6
数据结构跨缓存行 → 额外一次 cache miss

false sharing 检测: 两地址同一缓存行?
!(a >> 6 ^ b >> 6) → 同一缓存行
⚡ 第五部分：位传播原理 & nlz 关联 (6个)
PROP 01

并行前缀 OR 思想⭐⭐⭐

5步覆盖 = O(log n) 替代 O(n)
Guy Steele 贡献的核心思想
点击查看答案
并行前缀 (Parallel Prefix)
第k步覆盖 2ᵏ 个连续位
步k=5: 覆盖 32 位 ≥ 最高位+1

同构: 归并排序/Carry-Lookahead Adder/GPU前缀和
"Wherever you see parallel prefix, the material is due to Guy." —Warren
PROP 02

传播结果形态 ⭐⭐

传播后 = 2ⁿ - 1 形式
从最高1-bit到bit0全为1
点击查看答案
传播后形成"全1"掩码
任何数传播后都是 0...01...1 形式
例: 0x00100000 → 0x001FFFFF

这正是 2ⁿ-1 = 2¹·1111...1 的位模式
再 +1 就得到 clp2，再 -(x>>1) 就得到 flp2
NLZ 01

ilog2(x) = 31 - nlz(x) ⭐⭐

flp2(x) = 1 << ilog2(x)
整数 log₂ 直接给出位位置
点击查看答案
通过 ilog2 实现 flp2
ilog2(8) = 3, flp2(8) = 1<<3 = 8
ilog2(9) = 3, flp2(9) = 1<<3 = 8

int ilog2(unsigned x) {
return 31 - __builtin_clz(x); }
相当于找最高1位的位置
NLZ 02

pop 实现 flp2 ⭐

传播后 pop(x) = 位宽
flp2 = 1 << (pop-1)
通过 popcount 找最高位位置
点击查看答案
pop(x) 路径
传播后 x = 0x001FFFFF → pop(x) = 21
flp2 = 1 << (21-1) = 1 << 20 = 0x00100000

不如 nlz 方法高效，但理论价值高
注脚: "pop(x) is the number of 1-bits in x" —Warren
PROP 03

指令计数对比 ⭐⭐

flp2: 12条 | clp2: 13条
basic RISC 无分支实现
点击查看答案
flp2 vs clp2 指令分解
flp2: 5×OR + 5×SHR + 1×SHR + 1×SUB = 12
clp2: 1×SUB + 5×OR + 5×SHR + 1×ADD = 13

有 nlz/CLZ 硬件时: 两者都降为 2~3 条
PROP 04

Buddy System 应用⭐⭐⭐

buddy = pfn ^ (1 << order)
Linux 内存伙伴系统核心
点击查看答案
Linux Buddy System
order = ilog2(clp2(n));
分配 2^order 页的内存块
伙伴地址: pfn ^ (1 << order)

XOR 翻转第 order 位 = 跳到伙伴块
这就是 clp2 在 OS 内核里的核心应用
💡 记忆技巧
1. 对齐公式记忆口诀
向下：AND 负数 x & (-d)
向上：加 d-1 再 AND 负数 (x+d-1) & (-d)
记忆: "下去直接AND，上去先加垫脚石"
2. -d 为什么是掩码
d=4096 → 二进制低12位全0
-d (二补码取负) = 高位全1低12位全0
二补码性质使 -d 天然变成对齐掩码
3. flp2 = 传播 + 减去半身
5步OR-Shift把最高1位"感染"所有低位
再用 x - (x>>1) 删掉右边的"分身"
如同病毒复制后只留零号病人
4. clp2 的减1妙手
x是2次幂时: 减1后传播不会到达最高位
x不是2次幂时: 减1不影响最终结果
减1 = "我已在边界上"的特判消除技
5. XOR 跨越检测原理
同一2ᵏ块内所有地址高k+1位相同
a XOR (a+l-1) → 不同的位变1
右移k位后非零 = 高位不同 = 跨越了
6. 三个公式的统一视角
本质都是"x在2次幂网格中的位置"：
对齐 = 强制落在网格点
flp2/clp2 = 找最近的网格点
跨越检测 = 判断两点在同一格内
